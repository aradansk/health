> Per-phase validation contract for feedback sampling during execution.
> Source: `01.0-RESEARCH.md` → "## Validation Architecture". This is a 2-day
> timeboxed **spike** with binary pass/fail success criteria — there is no
> formal test framework; the spike script itself is the test.

---

## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | None formal — assertive Python script (`spike/spike_ingest.py`) with explicit `assert` + process exit codes. No pytest harness ships from a spike. |
| **Config file** | none — see Wave 0 |
| **Quick run command** | `python .planning/phases/HEALTH-01.0-fasten-ingest-api-spike/spike/spike_ingest.py` |
| **Full suite command** | same (single script; exit 0 = all spike success criteria 1–4 met, non-zero = escalation note required per success criterion 5) |
| **Estimated runtime** | ~60 seconds (container already up; signin → token → POST → verify → re-POST probe → log grep) |

---

## Sampling Rate

- **After every task commit:** Run the quick command against the live throwaway Fasten container (it is the only test).
- **After every plan wave:** n/a — single-wave spike.
- **Before `/gsd:verify-work`:** `spike_ingest.py` exits 0 AND `docs/fasten-admin.md` committed AND PII-log decision recorded — OR an escalation note (success criterion 5) documents the blocker + chosen fallback rank.
- **Max feedback latency:** ~60 seconds

## Wave 0 Requirements

- [ ] `spike/compose.spike.yaml` — throwaway single-service Fasten, digest-pinned, named volume, `:8080`, `FASTEN_JWT_ISSUER_KEY` override
- [ ] `spike/spike_ingest.py` — signin → mint `access` token → build+validate 1-resource Bundle (`fhir.resources`) → POST `/api/secure/source/manual` → verify via `GET /api/secure/resource/fhir` → re-POST idempotency probe → PII canary grep helper
- [ ] `spike/sample_bundle.json` — 1 Observation, **SYNTHETIC canary values only** (sentinel `ZZZ-PII-CANARY-0001`; CLAUDE.md PII Tier 1 rule — never real health data)
- [ ] `docs/fasten-admin.md` — THE durable deliverable: endpoint, auth (signin + `access` token), request shape (`multipart file=`), response envelope (`{success,data,source}` / 500), pinned digest, `jwt.issuer.key` security note, `LOG_LEVEL` decision, fallback ranking
- [ ] Framework install: none required (host Python 3.12 + `pip install "httpx>=0.27" "fhir.resources>=8.0.0"` — spike-local, no production manifest impact)
