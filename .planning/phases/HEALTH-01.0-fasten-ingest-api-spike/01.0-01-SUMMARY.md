## Deviations from Plan

### Auto-fixed Issues (RESEARCH.md assumptions corrected in-line — the spike's designed 30-min OQ iterations, not blockers)

**1. [Rule 3 - Blocking] STANDBY mode / missing encryption key**
- **Found during:** Task 2 (container boot).
- **Issue:** Pinned digest ships `database.encryption.enabled:true` with no key → Fasten boots in STANDBY, serving only the first-run wizard SPA + `/api/health`; REST API not reachable. Not in RESEARCH.md (which used `config.go` source defaults, not the shipped image config).
- **Fix:** Added `spike/config.spike.yaml` with a throwaway `database.encryption.key`; bind-mounted to `/opt/fasten/config/config.yaml`. `standby_mode` flipped to false; API became reachable.
- **Commit:** 1feb50d

**2. [Rule 3 - Blocking] Env-vars not honoured by pinned digest (OQ2)**
- **Found during:** Task 2. `FASTEN_JWT_ISSUER_KEY`/`FASTEN_LOG_LEVEL` did not override nested config. Used the RESEARCH.md-anticipated fallback: bind-mounted `config.yaml`. Documented as the working form for Phase 1.4.
- **Commit:** 1feb50d

**3. [Rule 1 - Behavior] HTTPS by default + base path / signup-then-signin**
- **Found during:** Task 2. Shipped config has `web.https.enabled:true` (self-signed); REST base path is `/api` (the `/web/api` seen earlier was a STANDBY SPA artifact); first user must be created via `/api/auth/signup` before signin. Disabled HTTPS in spike config; updated `spike_ingest.py` to signup-then-signin and HTTP base. Resolved OQ1: a `Bundle{type:"collection"}` was accepted as-is (no Fasten-export profiling).
- **Commit:** 1feb50d

**4. [Documented finding] INFRA-06 digest drift**
- Pinned `sha256:3f0192ac…` is now the per-arch amd64 manifest, not the live `:main` OCI index (`sha256:d208351137…`) — the floating-tag drift the pin exists to catch (Pitfall 3 validated). Both digests recorded in `docs/fasten-admin.md` §2.1; Phase 1.4 must consciously choose + re-verify. No silent re-pin.

**5. [Environment] Host port remap** — host 8080 and 18080 occupied; spike mapped host **18081 → container 8080**. Documented in compose + `docs/fasten-admin.md`.
